## 3. Spread / Total Edge 偵測（最大功能）

### 關鍵 API 發現
`marketGroup` 60067.1 **只回傳 `不讓分`（moneyline）**。讓分 / 大小分在 **`type: "event"`** 端點。

```python
POST /services/content/get
body: {"contentId": {"type": "event", "id": "3461352.1"}, ...}
# 回傳 26 個 markets: 不讓分、[總分]讓分 X (3條線)、[總分]大小 X、半場/單節等
```

探索腳本 `src/api_explore_markets.py` 記錄了整個探索過程（嘗試 marketGroup 60067.2~60075.1 都失敗，最後用 event ID）。

### 資料流
```
fetcher.py:
  1. fetch marketGroup 60067.1 → 取 event IDs + moneyline
  2. 對每個 event ID 呼叫 fetch_event_detail() → 取所有 markets
  3. parse_event_markets() 只取 [總分]讓分 和 [總分]大小（忽略半場/單節）
```

### Parser 變更
`api_parser.py`：
- 新增 `parse_event_markets(data, existing)` — 合併到現有 GameOdds
- 市場 prefix 匹配：`FULL_GAME_SPREAD = "[總分]讓分"`、`FULL_GAME_TOTAL = "[總分]大小"`
- 線值從 market name 提取（regex `([+-]?\d+(?:\.\d+)?)`）

### Edge 偵測演算法
`edge_detector.py`：

```python
# 讓分覆蓋機率（常態模型）
P(away covers -line) = Φ((-line - pred_spread) / σ_spread)
σ_spread = 12 pts (NBA 歷史單場分差標準差)

# 大小分
P(over line) = Φ((pred_total - line) / σ_total)
σ_total = 18 pts
```

- `detect_spread_edges()` — 每場比賽 × 每條讓分線 × 2 方向（home/away）
- `detect_total_edges()` — 每場 × 每條總分線 × over/under
- Edge dataclass 加 `edge_type` 和 `line` 欄位
- 主流程合併三種：`edges = ml_edges + sp_edges + tt_edges`

### 實測結果
2 場比賽 → **10 個 edges**：🎰 獨贏 2 · 📐 讓分 6 · 📊 大小 2

| # | 推薦 | Edge |
|---|---|---|
| 1 | 🎰 Phoenix Suns | +40.5% |
| 2 | 📐 Suns +4.5 | +25.8% |
| 3 | 📊 Over 217.5 | +24.3% |

## 4. 歷史回測分析

### Outcome Resolver
`src/sport_resolve.py` — 從 ESPN API 拉已完成比賽比分，三種 edge 的勝負判定：

```python
moneyline: winner_side == edge.side
spread:
  away covers -line: away_score - home_score > line
  home covers +line: home_score - away_score > -line
total:
  over:  home_score + away_score > line
  under: home_score + away_score < line
```

每輪 edge_detector 執行前會自動 resolve 過去未解析的 edges。

### 分析引擎
`sport_db.py` → `edge_backtest()`：

| 維度 | 欄位 |
|---|---|
| 總覽 | total / resolved / won / lost / win_rate / total_profit / avg_roi |
| 按類型 | 獨贏 / 讓分 / 大小 各自勝率和損益 |
| Edge 大小分層 | 0-5% / 5-10% / 10-20% / 20%+ |
| 模型校準 | model_prob 40-50% / ... / 80%+ vs 實際勝率 |
| 最近結果 | 最新 20 筆（✅/❌ + 損益） |

### Streamlit 顯示
`quantSignal_clone/streamlit_app/app.py` 運彩 Edge tab 底部加 `📈 Edge 歷史回測` 區：
- 4 格 metric：已驗證 / 勝率 / 累計損益 / 待驗證
- 按類型 3 張卡
- Edge 大小分層表
- 模型校準表（差距 > 0 = 模型保守，< 0 = 過度自信）
- 最近已驗證 edge 列表

## 已知問題 / 待優化

1. **edges 表沒 UNIQUE constraint** — 每次執行都累加，需要加 `UNIQUE(detected_at, game_id, side, edge_type, line)` 去重
2. **σ 為固定值** — 讓分 σ=12、大小 σ=18 是歷史均值，季後賽可能需調整
3. **pred_spread 與 home_prob 偶有矛盾**（例：Orlando 52.1% 勝率 + pred_spread=-10）— 可能因為兩個模型獨立訓練
4. **Kelly 爆表**（Suns ML Kelly 92.5%）— 模型過度自信的警訊，實盤應用 1/4 Kelly
