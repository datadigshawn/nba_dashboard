## 1. 季後賽 Bracket 視覺化

### 目標
使用者給了一張 NBA 季後賽對戰樹設計圖（7 欄：R1 → R2 → CF → Finals → CF → R2 → R1），要做出來。

### 資料層（`nba_predictor.py`）

新增常數：
- `CONFERENCE_MAP` — 30 隊分東西區
- `TEAM_ABBREV` — 隊名縮寫
- `TEAM_STARS` — 每隊招牌球員（Wemby、SGA、Luka、Jokic 等）

新增函式：
- `fetch_espn_injuries()` — 從 `https://...nba/injuries` 抓傷病清單
- `_elo_game_prob(elo_a, elo_b, hca=48)` — Elo 對決每場勝率
- `_series_expected_games(p, best_of=7)` — 系列賽預期場數（解析式）
- `_simulate_series()` — 單次 best-of-7 模擬（2-2-1-1-1 home pattern）
- `build_playoff_bracket()` — **10,000 次 Monte Carlo**，算各隊在每輪的挺進機率

### 資料結構
```json
{
  "playoff_bracket": {
    "generated_at": "...",
    "n_sims": 10000,
    "west": {
      "seeds": [8 支隊伍],
      "r1": [4 個對戰],  // 1v8, 4v5, 3v6, 2v7
      "r2": [2 個對戰],  // 半區冠軍
      "conf_finals": [1 個對戰]
    },
    "east": {同上},
    "finals": {"west": {...}, "east": {...}, "expected_games": X}
  }
}
```

每張 card 含：
- 種子 + 縮寫 + Elo + 戰績
- `advance_prob` (MC 模擬的挺進機率)
- `star`: {name, status (ESPN injuryStatus), detail}

## 2. SQLite 資料庫

### 動機
nba_data.json 每天覆蓋，歷史預測全部丟失。要能回測預測準確率、追蹤 Elo 漂移、校準模型信心。

### 檔案
- `nba_db.py` — DB 模組（130 行）
- `nba_backfill.py` — 一次性回填 nba_data.json 現有資料
- `nba_resolve.py` — 每日執行，填入昨日比賽結果
- `nba.db` — SQLite（加入 .gitignore）

### Schema
```
predictions(prediction_date, game_date, home, away, home_prob, away_prob,
            home_elo, away_elo, pred_spread, pred_total, ...,
            home_score, away_score, winner, pick_correct, margin_error, resolved_at)
elo_history(date, team, elo)
daily_performance(date, games_tested, all_wr, strong_wr, vstrong_wr, ...)
backtest_results(run_date, game_date, home, away, conf, pick, winner, correct, score)
```

所有表有 UNIQUE constraint，`INSERT OR IGNORE` 冪等。

### Pipeline 接點
- `nba_predictor.py` 的 `--json` 路徑最後自動寫 DB（try/except 不影響 JSON 管線）
- `nba_daily_update.sh` 加 step 5：`nba_resolve.py`（每天跑一次填昨日結果）

### 回填結果
- predictions: 10 筆（今日 13 場，3 場因 play-in 合併隊名衝突被去重）
- elo_history: 15 隊（從 nba_data.json top 15）
- daily_performance: 1 筆
- backtest_results: 30 筆（從 nba_data.json 的 backtest.recent）
- resolve: 嘗試填 198 場 ESPN 歷史結果（因 predictions 只有今日，所以 0 筆 match）

## 自動化流程

```
每天 09:00 (com.nba.daily_update)
  nba_daily_update.sh:
    1. 訓練 XGBoost 主模型
    2. 訓練 Spread 模型
    3. --json --days-ahead 3 → nba_data.json (含 playoff_bracket)
    4. → 自動寫 nba.db (predictions + elo + backtest)
    5. nba_resolve.py → ESPN 查昨日結果 → 填 DB outcome
    6. sync_data.py → GitHub Release → Streamlit Cloud
```
